# ホワイトニングサロン「プラン診断」完成報告

仕様書に基づき、単一の `index.html` ファイルで構成されたLIFF診断アプリを実装しました。

## 実装内容（更新）

### 1. 診断フロー
- **Intro画面**: アプリの目的を伝え、診断を開始します。
- **質問画面 (Q1-Q4)**: スムーズなアニメーションと共に質問を表示し、プログレスバーで進捗を可視化します。
- **分析中画面**: 分析時間を設けることで期待感を高めます。
- **結果画面 (NEW)**:
    - 診断結果が表示されると同時に、**自動的にLINEトークへ結果を送信**します。
    - 画面下部に「診断を完了する」ボタンを配置し、タップでLIFFウィンドウを閉じられるようにしました。

### 2. 機能
- **自動送信機能**: ユーザーの手間を減らすため、結果画面遷移時に `liff.sendMessages` と GASへのPOSTを自動実行します。
- **LIFF連携**: `liff.init` による初期化、プロファイル取得、および `liff.closeWindow` によるクローズ処理。
- **データ保存**: 診断結果をGAS（Google Apps Script）のWebhookへ自動送信。

### 3. デザイン
- 清潔感のあるブルー（`#6bc1ed`）を基調としたモダンなデザイン。
- 結果送信中や送信完了の状態を画面下部にテキストで表示。

## ファイル構成
- `index.html`: 全てのHTML、CSS、JavaScriptが含まれる単一ファイル。

## 使用方法
1. LIFF ID: `2005378903-eFXXT5Gj` が既定されています。
2. GASのデプロイ後、`index.html` 内の `CONFIG.GAS_URL` を実際のエンドポイントに書き換えてください。
